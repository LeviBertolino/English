# Unit 025 — have something done

## 0. 🔁 Warm-up: Quick Recall (≈60 seconds)

Yesterday's lesson — **Unit 024: is being done / has been done (Passive 2)** — covered two passive forms: present continuous passive for actions happening *right now* (*"The broken node **is being replaced**"*), and present perfect passive for recently completed actions whose result matters *now* (*"The customers **have been notified**"*, *"Has the incident **been logged**?"*). The memory tip was that the object takes the spotlight, and passive is completely natural in incident reports and process language.

Three rapid recall questions:

1. Which is correct? *"The node **is replaced** right now"* OR *"The node **is being replaced** right now."*
2. Transform to present perfect passive: *"Someone has deployed the fix."* → *"The fix ___ ___ deployed."*
3. True or false: In passive voice, the agent (*by someone*) must always be stated.

👉 **Answers:**
1. ✅ *"The node **is being replaced** right now."* — ongoing action needs *being*
2. ✅ *"The fix **has been** deployed."*
3. ✅ **False** — the agent is usually dropped when it's obvious or unimportant

## 2. 🧠 Adquirir — Mini-context first, rule second

> **A 1:1 between Levi and his manager, mid-migration:**
>
> **Manager:** How's the database migration going?
>
> **Levi:** Good — we **had it reviewed** by the DBA team before we touched anything.
>
> **Manager:** Smart. Are you doing the actual cutover yourselves?
>
> **Levi:** No, we're **getting it handled** by the platform team — they have more experience with zero-downtime cutovers.
>
> **Manager:** And the rollback plan?
>
> **Levi:** We **had it tested** twice in staging already. If it fails, we'll **get it fixed** before we try again.

Notice:
- *"We **had it reviewed**"*, *"getting it handled"*, *"had it tested"*, *"get it fixed"* — in every case, **someone else** does the action, but the subject (*we*) arranged it or caused it to happen.
- The pattern is always: **subject + have/get + object + past participle**. The object sits right after *have/get*, then the past participle follows.

### The rule

Use **have/get + object + past participle** when you arrange for someone else to do something for you — you don't do it yourself, but you cause it to happen.

| Structure | Example |
|---|---|
| **have** + object + past participle | *"We **had** the pipeline **migrated** by the platform team."* |
| **get** + object + past participle | *"I'm **getting** my laptop **repaired** this week."* (more informal/casual than *have*) |

> ⚠️ Common error: don't use an infinitive here. ❌ *"We had the DBA **to review** it."* → ✅ *"We had it **reviewed** by the DBA."* The verb after the object must be a **past participle**, not *to + verb*.

**Compare with passive voice (Unit 023–024):**
- *"The code **was reviewed**."* — passive: just states the action happened, no focus on who arranged it.
- *"We **had** the code **reviewed**."* — causative: emphasizes that **you** arranged/caused the review to happen.

**Bonus meaning — something happens to you (often unfortunate):**
*Have/get + object + past participle* can also describe an experience that happened to you, without you arranging it:
- *"We **had** our servers **hacked** last month."*
- *"They **had** their funding **cut**."*

## 5. 💡 Memory tip

**HAVE = hand it off.** If you didn't do it with your own hands, you *hand* the task to someone else: **have/get + [the thing] + [done to it]**. Think of it like packing labels — the object gets the "done" sticker, not the person doing it.

## 8. 🇧🇷 Resumo em português

A estrutura **have/get + objeto + particípio passado** (ex.: *"we had it reviewed"*, *"I'm getting my laptop repaired"*) é usada quando alguém **organiza ou manda outra pessoa fazer algo por você** — você não faz a ação, mas é responsável por ela acontecer. *Have* é mais neutro/formal; *get* é mais informal. Repare que depois do objeto vem sempre o **particípio passado**, nunca o infinitivo — errado seria *"we had the DBA to review it"*; o certo é *"we had it reviewed by the DBA"*. Essa estrutura também pode descrever algo que **aconteceu com você**, geralmente algo ruim, sem que você tenha organizado (*"we had our servers hacked"*).

Evite confundir essa construção com a voz passiva simples: a passiva (*"the code was reviewed"*) só diz que a ação aconteceu; a estrutura causativa (*"we had the code reviewed"*) deixa claro que **você** foi quem organizou ou solicitou a ação. Preste atenção também na ordem das palavras: objeto sempre logo depois de *have/get*, seguido do particípio — inverter essa ordem é o erro mais comum entre falantes de português.
