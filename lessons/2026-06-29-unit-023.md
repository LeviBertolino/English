## 2. 🧠 Adquirir — Mini-context first, rule second

> **A post-incident Slack thread:**
>
> **@DevOps (Sasha):** Quick update on last night's outage. The root cause **was identified** — a misconfigured load balancer. The service **was restored** at 03:14 UTC and all affected customers **have been notified**.
>
> **@Levi:** Thanks. When **was** the alert **triggered**? And **is** the runbook **updated**?
>
> **@Sasha:** The alert **was triggered** at 02:47. The runbook **is being updated** now — done in an hour.
>
> **@Levi:** Good. **Is** every P1 incident **reviewed** by engineering leadership?
>
> **@Sasha:** Yes — all P1s **are reviewed** by the engineering leads within 48 hours. The template **was redesigned** last quarter so it's much cleaner now.

Notice:
- *"The root cause **was identified**"* → past simple passive (*was* + past participle)
- *"**Is** the runbook **updated**?"* → present simple passive question
- *"All P1s **are reviewed**"* → present simple passive (*are* + past participle) — ongoing policy
- The agent (*by someone*) is often left out when it's obvious or unimportant
- In tech and business writing, passive voice is natural and frequent — incident reports, documentation, process descriptions all rely on it

---

### The rules for Unit 023

#### Rule 1 — What is the passive?

In **active** sentences, the subject performs the action:
> *"The team deployed the service."* (team = subject doing the action; service = object receiving it)

In **passive** sentences, the subject **receives** the action instead:
> *"The service **was deployed**."* (service = subject, but it was acted upon — team is now optional or omitted)

**When to use passive:**
1. The **agent is unknown**: *"The bug was introduced in the last commit."* (we don't know who)
2. The **agent is obvious or unimportant**: *"The PR was merged."* (clearly a developer did it — the fact matters, not who)
3. You want to **focus on what happened**, not who did it: *"The database was corrupted during the migration."*

---

#### Rule 2 — How to form the passive

| Tense | Structure | Example |
|---|---|---|
| **Present simple** | am / is / are + past participle | *"The pipeline **is triggered** automatically."* |
| **Past simple** | was / were + past participle | *"The deploy **was approved** last night."* |

To say **who** did it, add **by**:
> *"The PR **was reviewed** **by** two senior engineers."*
> *"The service **is maintained** **by** the platform team."*

---

#### Rule 3 — Passive question and negative forms

| Form | Example |
|---|---|
| Question | *"**Is** the documentation **updated**?"* / *"**Was** the alert **triggered**?"* |
| Negative | *"The fix **isn't deployed** yet."* / *"The issue **wasn't caught** in QA."* |

> ⚠️ **Brazilian Portuguese note:** In PT-BR, passive is often expressed with *se apassivador*: *"Encontrou-se o bug"*, *"Aprovaram-se as mudanças."* In English, these constructions don't exist — the passive with *be + past participle* is the clean, standard equivalent. Avoid translating the *se* forms literally — they produce unnatural English.

## 3. ✏️ 8 Example Sentences

1. **"Every pull request is reviewed by at least two engineers before it's merged into main."** *(present simple passive — standard team policy)*

3. **"New features are documented in Confluence before they're released to production."** *(present simple passive — process description)*

7. **"My laptop is charged overnight — I always start the day with 100%."** *(present simple passive — everyday context)*
8. **"The last email was sent by mistake — please ignore it."** *(past simple passive + by-agent — everyday context)*

## 4. 🛠️ Practice — recognition + production

### Recognition — gap-fill

Transform each active sentence into a passive sentence using present simple or past simple:

1. *"The security team audits all code changes."* → *"All code changes ___ ___ (audit) by the security team."*
2. *"Someone designed the new API gateway in Q2."* → *"The new API gateway ___ ___ (design) in Q2."*
3. *"The team doesn't store passwords in plain text."* → *"Passwords ___ ___ (not/store) in plain text."*

✅ **Answer:**
1. *"All code changes **are audited** by the security team."*
2. *"The new API gateway **was designed** in Q2."*
3. *"Passwords **aren't stored** in plain text."*

## 5. 💡 Memory Tip

Think of passive voice as **"the object takes the spotlight."**

> Active: **The team** ← spotlight → deployed the service.
> Passive: **The service** ← spotlight → was deployed (by the team).

The structure is always: **subject + be + past participle**

Quick self-check pattern:
- *is/are* + past participle → **present simple passive** (policy, habit, general truth)
- *was/were* + past participle → **past simple passive** (completed at a specific past moment)

## 8. 🇧🇷 Resumo em português

A voz passiva em inglês é essencial para comunicação técnica e profissional. A estrutura básica é: **sujeito + verbo "to be" + particípio passado**. No presente simples, use *is/are*: *"The PR **is reviewed** by two engineers"* (regra geral, política da empresa, verdade recorrente). No passado simples, use *was/were*: *"The bug **was found** during QA"* (evento concluído num momento específico do passado). Para indicar quem realizou a ação, adicione *by*: *"The service **was deployed** by the DevOps team."* Na maioria das vezes o *by* é omitido porque o agente é óbvio ou irrelevante.

Um ponto importante para falantes de português: em PT-BR, o *se apassivador* (*"Encontrou-se o bug"*, *"Aprovaram-se as mudanças"*) ou o *se* reflexivo substituem a passiva em muitos contextos. Em inglês, essas construções não existem — a voz passiva com *be + particípio* é o equivalente direto e padrão. Além disso, a passiva é muito mais aceita e frequente no inglês técnico escrito do que o português sugere: documentação, relatórios de incidente, políticas de código, anúncios formais — tudo usa passiva de forma natural. Dominar *is done / was done* vai elevar imediatamente a qualidade do seu inglês profissional escrito.
