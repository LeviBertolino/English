From yesterday's examples: *"Does your CI pipeline run integration tests on every commit? Ours doesn't."* and *"She doesn't own a specific product area — she works across the entire platform."*

1. Fill in: *"\_\_\_ your team use TypeScript on the backend?"*
2. Correct or wrong? *"She doesn't reviews PRs daily."*
3. Short answer: *"Do you push directly to main?"* → *"No, I \_\_\_."*

👉 **Answers:**
1. **Does** — "your team" is third-person singular.
2. ❌ Wrong — when **doesn't** is present, the main verb stays base form: *"She doesn't review PRs daily."*
3. **don't** — short negative answer mirrors the auxiliary: *"No, I don't."*

## 2. 🧠 Adquirir — Mini-context first, rule second

> **Slack message — #incident-channel:**
>
> *Someone: "Hey Levi, can you jump in? Prod is throwing 500s."*
>
> *Levi: "On it. I'm looking at the traces right now."*
>
> *Someone: "Is the on-call engineer checking the database too?"*
>
> *Levi: "Yes — Ana is investigating the slow queries. We're also watching the error rate in Datadog. It's dropping, so I think we're making progress."*
>
> *Someone: "What are you doing about the retry storms?"*
>
> *Levi: "I'm adjusting the backoff config as we speak."*

Read again: **I'm looking / is checking / we're watching / I'm adjusting**. These describe things **happening right now, at this moment** — not general habits or permanent facts. That's the present continuous.

### The rule

Use the **present continuous** (also called present progressive) for:

1. **Something happening right now, at this moment:**
   - *"I'm looking at the traces."* (I opened the dashboard; I'm doing it now)
   - *"The build is failing."* (happening as we speak)

2. **Something happening around now — this period — even if not at this exact second:**
   - *"We're migrating our infrastructure to Kubernetes this quarter."* (a current, ongoing project)
   - *"She's leading the AI safety initiative."* (her current focus area, not necessarily at this second)

#### How to form it

| Person | Structure | Example |
|---|---|---|
| I | am + verb-ing | *I am reviewing the PR.* |
| he / she / it | is + verb-ing | *She is deploying the hotfix.* |
| you / we / they | are + verb-ing | *They are running load tests.* |

**Contractions** (natural in speech and Slack):
- I am → **I'm** / he is → **he's** / she is → **she's** / we are → **we're** / they are → **they're**

#### -ing spelling notes

| Rule | Example |
|---|---|
| Most verbs: add -ing | work → **working**, check → **checking** |
| Verb ends in -e: drop -e, add -ing | write → **writing**, take → **taking** |
| Short verb ending in consonant-vowel-consonant: double last consonant | run → **running**, debug → **debugging**, get → **getting** |

## 3. ✏️ 8 Example Sentences

1. **"I'm reviewing the architecture proposal right now — can we talk in 20 minutes?"** — present continuous, right now, engineering context
2. **"The team is onboarding three new engineers this sprint, so capacity is tighter than usual."** — around now/current period, team management
3. **"Our platform is handling more traffic than expected — the autoscaler is kicking in."** — happening now, infrastructure/SRE
4. **"She's leading the migration to the new data lake — it's her main focus this quarter."** — current ongoing project, Staff engineering
5. **"We're evaluating two LLM providers this month to decide which to use in production."** — around now, AI/tech leadership
6. **"The model is processing the batch job — it should finish in about 10 minutes."** — happening now, AI/ML
7. **"I'm cooking dinner while I wait for the deployment to finish."** — right now, everyday life
8. **"They're renovating the office this week, so everyone is working from home."** — around now, everyday context

## 4. 🛠️ Practice — recognition + production

### Recognition — gap-fill

Complete the status update in a stand-up meeting. Use the correct present continuous form:

> *"This week I \_\_\_ (work) on the new API gateway spec. Priya \_\_\_ (pair) with me on the authentication layer. We \_\_\_ (not / block) anyone else, but we \_\_\_ (wait) for the security team's sign-off before we ship."*

✅ **Answer:**
1. **am working** — I + am + verb-ing
2. **is pairing** — Priya (she) + is + verb-ing; *pair* ends in -r, short CVC → double the r: *pairing*
3. **are not blocking** (or **aren't blocking**) — we + are + not + verb-ing
4. **are waiting** — we + are + verb-ing

### Production task

*Write 2 sentences about something you or your team is currently working on — one about "right now at this moment" and one about "this week/quarter."*

**Model answer:**
1. *"I'm reading a system design proposal right now — I'll give feedback before end of day."*
2. *"Our team is redesigning the incident response playbook this quarter to reduce MTTD."*

## 5. 💡 Memory Tip

**Think of the -ing as a "live indicator"** — like a blinking dot on a status dashboard. When you see or use **am/is/are + verb-ing**, picture that dot blinking: *something is active right now or in this window of time.*

- No blinking dot (present simple): *"We deploy on Fridays."* (a fixed schedule — it's always true)
- Blinking dot (present continuous): *"We're deploying right now."* (it's happening — watch the pipeline)

The blinking-dot image also helps you remember the **double consonant** rule: *run → running*, *debug → debugging* — the double letter "keeps the dot steady" (the vowel sound stays short).

## 8. 🇧🇷 Resumo em português

O **presente contínuo** (present continuous) é formado com **am/is/are + verbo-ing** e descreve duas situações: algo que está acontecendo **agora mesmo** (*"I'm looking at the traces right now"*) ou algo que está em andamento **neste período** — esta semana, este mês, este trimestre (*"We're migrating to Kubernetes this quarter"*). A grande diferença em relação ao presente simples é essa: o presente simples descreve hábitos e verdades permanentes (*"We deploy on Fridays"*), enquanto o contínuo indica atividade ativa e temporária (*"We're deploying right now"*).

Atenção às regras de ortografia do **-ing**: a maioria dos verbos recebe -ing normalmente (*working*, *checking*), verbos terminados em -e perdem o e (*write → writing*), e verbos curtos com padrão consoante-vogal-consoante duplicam a última consoante (*run → running*, *debug → debugging*, *get → getting*). Para brasileiros, o erro mais comum é usar o presente simples onde o inglês pede o contínuo — especialmente em atualizações de status. Se algo está acontecendo agora ou nesta fase do projeto, use **am/is/are + -ing**, não o verbo simples.
